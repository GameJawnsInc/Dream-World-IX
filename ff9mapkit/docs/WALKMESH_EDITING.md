# Spec — editing an imported multi-floor walkmesh without losing connectivity

Status: **v1 shipped** (verbatim ship + build-time guard). v2/v3 designed below, not yet built.

## 1. The problem

`ff9mapkit import --editable` forks a real field. Its background art splits into editable layers
cleanly. Its **walkmesh** does not — if we re-export it through Wavefront `.obj`, multi-floor fields
come back **disconnected** (the player gets trapped on one floor; observed in-game on Gargan Roo,
7 floors). Root cause, stated precisely:

- The `.bgi` **codec is lossless** — `BgiWalkmesh.from_bytes`/`to_bytes` round-trips any real walkmesh
  byte-for-byte.
- The `.obj` **intermediate is lossy** — a Wavefront `.obj` carries only **geometry** (verts, faces,
  the `o` floor grouping). It cannot carry the **navmesh adjacency graph**: per-triangle neighbor
  links (`nbr`/`edge`/`edgeClone`), edge flags (one-way / blocking / footstep), floor flags, or
  animations (moving platforms).
- On build, `bgi.build` *reconstructs* adjacency by linking triangles that **share a vertex index**.
  That is correct *within* a floor, but FF9 gives **each floor a disjoint vertex set**, so cross-floor
  triangles share no index → no link → floors strand.

The deeper truth: **a navmesh's connectivity is not a function of its geometry.** Two floors can be
geometrically touching but walled off, or geometrically separated but joined by a ladder/step/warp.
FF9 stores that adjacency *explicitly*. No heuristic recovers it from vertex positions alone. So a
geometry-only round trip **cannot** be lossless for a hand-authored multi-floor mesh — this is a
property of the representations, not a fixable bug.

## 2. What must survive an edit (beyond geometry)

| data | in `.bgi` | derivable from geometry? |
|---|---|---|
| verts, faces, floor partition | yes | **yes** (`o` groups) |
| triangle centers, plane normal/`d` | yes | yes (recompute) |
| intra-floor neighbor links | `tri.nbr`/`edge`/`edgeClone` | yes (shared vertex **index** within a floor) |
| **cross-floor neighbor links** | `tri.nbr` across floors | **NO** — disjoint vertex sets |
| **edge flags** (one-way, blocking, footstep) | `edge.flags` | **NO** |
| **floor flags**, `floor.org`/`cur` (moving platforms) | `floor.*` | **NO** |
| **animations** (moving-platform keyframes) | `anms` | **NO** |
| `charPos`, `activeFloor`/`Tri` | header | **NO** (cheap to carry) |

Everything in the **NO** rows is what a sidecar must preserve.

## 3. Design — geometry (`.obj`) + adjacency sidecar (`walkmesh.links.toml`)

Keep the geometry in `.obj` (so Blender can edit it). Put everything the `.obj` can't express in a
sidecar, **keyed by world position** so it survives Blender renumbering. `.bgi` verts are `Int16`, so
positions are exact integers — a key is an **exact** match, no float epsilon.

### 3.1 Sidecar format

```toml
# walkmesh.links.toml — auto-generated by `ff9mapkit import --editable`.
# Edited geometry (walkmesh.obj) reconciles against these by world position on build.
[header]
active_floor = 0
active_tri   = 12
char_pos     = [404, -2135, 127]

# A cross-floor connection geometry can't express. Endpoints are world positions.
[[seam]]
a_floor = 0
a_edge  = [[-120, -2135, 340], [80, -2135, 340]]
b_floor = 1
b_edge  = [[-120, -2135, 340], [80, -2135, 340]]   # == a_edge → floors physically meet (coincident)
# A ladder/step/warp seam instead has a_edge != b_edge (two distinct edges to bridge).

# Non-default edge semantics (one-way direction, blocking, footstep sound).
[[edge_flag]]
floor = 2
edge  = [[1100, -1800, -50], [1100, -1800, 90]]
flags = 4

# Moving-platform animation, carried verbatim; identified by triangle centroids.
[[anim]]
floor  = 3
tris   = [[cx, cy, cz], ...]
frames = [ ... ]
```

### 3.2 Export (automatic — we hold the original `.bgi`)

`import --editable` reads the original `.bgi`, so it emits the sidecar with **zero guessing**:

1. Write `walkmesh.obj` (world verts, one `o floor_N` per floor).
2. For **every** cross-floor `nbr` link in the original (coincident *or* bridge), emit a `[[seam]]`
   keyed by the two tris' edge endpoint positions. (Reading the original's links captures ladders and
   warps that position-coincidence alone would miss.)
3. Emit `[[edge_flag]]` for every non-default `edge.flags`; `[[anim]]` for each animation (tri
   centroids as keys); `[header]` from the original.

Export is therefore lossless: the `.obj` + sidecar together carry the whole original `.bgi`.

### 3.3 Build (reconciliation) — `resolve_walkmesh` when `[walkmesh] obj + links =`

```
load walkmesh.obj            → verts, faces, floor groups
per floor: rebuild_neighbors by shared vertex INDEX     # intra-floor links (safe)
for seam in sidecar:
    a = find_tri_edge(floor=seam.a_floor, endpoints=seam.a_edge)   # match by position
    b = find_tri_edge(floor=seam.b_floor, endpoints=seam.b_edge)
    if a and b: link(a, b)            # set nbr + edgeClone on both slots
    else:       warn(f"seam broken near {seam.a_edge}: edge moved/deleted — re-anchor it")
apply edge_flag overrides (position-keyed)
re-attach anims by tri-centroid match (warn + drop on miss)
recompute centers/normals/d/min/max/dataSize; set header from [header]
assert reachability (section 4); warn on stranded floors
```

`find_tri_edge` matches an unordered endpoint pair against each triangle's three edges within the
named floor.

### 3.4 Failure modes (graceful, never silent)

| edit | result |
|---|---|
| unchanged geometry | identical to original (every seam/flag/anim matches) |
| reshape a floor's **interior** (not its seam edges) | fully preserved |
| **move/delete a seam edge** | that seam fails to match → **explicit warning** naming the world position to re-anchor; link dropped (reachability then flags it) |
| **add a floor** | it's an island until you add a `[[seam]]`; the tool can *suggest* seams where the new floor's edges coincide with an existing floor's |
| reshape an **animated** floor | anim centroid match fails → warn + drop the anim |

The guarantee is **no silent mis-link**: anything we can't reconcile is reported, and the reachability
check (below) catches the consequence at build time.

## 4. Build-time verification — reachability (shipped in v1)

`BgiWalkmesh.reachable_floors()` BFS's the triangle `nbr` graph from the start tri — the *same* links
the engine pathfinds over. `build` warns when `all_floors() - reachable_floors()` is non-empty:

```
walkmesh: floor(s) [1, 2, 3, 4, 5, 6] not walk-reachable from the start (6 of 7 floors stranded).
If you re-exported a multi-floor walkmesh via [walkmesh] obj, cross-floor links were lost — ship the
original with [walkmesh] bgi, or declare seams (docs/WALKMESH_EDITING.md).
```

This turns the in-game "trapped on one floor" symptom into a **build-time** warning. (Tests:
`test_obj_reexport_loses_cross_floor_connectivity`, `test_build_warns_on_stranded_floors`.)

## 5. Tooling

- **extract** (v2): write `walkmesh.links.toml` alongside `walkmesh.obj`; `field.toml` emits
  `[walkmesh] obj = "walkmesh.obj"` + `links = "walkmesh.links.toml"`.
- **build** (v2): the reconciliation in 3.3 behind that `[walkmesh] obj + links` form.
- **`ff9mapkit walkmesh verify <bgi>`** (v3): reachability + dangling-link report as a standalone check.
- **Blender** (v3): draw seam edges as a marked edge set; a "re-anchor seam" operator; surface the
  stranded-floor warning in the panel. Investigate Blender **custom face attributes** as a more
  robust key than position for the in-Blender path (they survive many edits that move vertices).

## 6. Incremental delivery

- **v1 — DONE.** `[walkmesh] bgi` ships a real walkmesh **verbatim** (forks are lossless); build-time
  reachability warning; tests guard the obj-reexport loss. This is enough to fork *and keep* any
  multi-floor field — you just can't reshape its geometry yet.
- **v2.** Auto-export the seam sidecar + build reconciliation by position + warn-on-broken-seam.
  Unlocks editing the *interior* geometry of a multi-floor fork while keeping connectivity.
- **v3.** Edge-flag + anim carry; the `walkmesh verify` CLI; Blender seam viz + re-anchor + "suggest
  seams" for newly-added floors.

## 7. Non-goals / open questions

- **Auto-inventing** connectivity for brand-new floors — you declare the seam; we can only *suggest*
  coincident ones.
- **Authoring** moving-platform animations — v3 carries them verbatim, it doesn't create them.
- Whether to key the Blender path by world position or by a **custom face attribute** Blender
  preserves through edits (more robust, but Blender-specific) — decide during v3.
- Semantic sanity ("does this ladder make sense") is out of scope; the engine doesn't care and neither
  do we.

## TL;DR

The codec is lossless; `.obj` is not, because adjacency isn't geometry. For **forking**, ship the
original `.bgi` verbatim (v1, done) — zero loss. For **reshaping** a multi-floor fork, pair the `.obj`
with a position-keyed adjacency **sidecar** (v2/v3) that reconciles against your edits and warns,
never mis-links. The reachability guard makes the failure loud at build time instead of in-game.
