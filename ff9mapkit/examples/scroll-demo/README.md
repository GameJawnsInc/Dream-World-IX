# Scrolling demo — a larger-than-screen room

A worked example of a **scrolling field**: the painting is 2× the screen wide, and the view pans
to follow the player. In-game-verified (field 4003).

```bash
ff9mapkit build scroll_demo.field.toml --out dist --mod-name ScrollDemo
```

## What it shows
- `[camera] range = [768, 448]` + `window_width = 384` — a 2×-wide painting at a normal focal length.
- `[camera.scroll] enabled = true` — the kit auto-sets the scroll bounds and injects the engine's
  `EnableCameraServices`, so Memoria's built-in 3D scroll pans the view. (Runs on stock Memoria.)
- A `back.png` (floor + walls, the whole scene) + a `front.png` foreground occluder (the hanging
  lamp, `z = 8`, draws over the player).
- A flat walkmesh spanning the painting, with `[walkmesh] frame = "world"` — the quad's 4 corners are
  true world coords (= the painted floor), so the character plants with no offset.

## The paint guide
`art/paint_template.png` / `art/paint_guide.png` were generated by `tools/make_scroll_demo.py`.
They size to the **full painting** and include **height guides** (vertical poles + back-wall rings +
a room-box outline, labeled in world units) so walls can be painted in correct perspective — not
just a floor grid. Paint `art/back.png` over the template, then build.
